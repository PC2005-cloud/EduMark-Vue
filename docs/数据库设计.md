# 数据库设计

## 1. 概述

本文档定义 EduMark 作业批改系统的数据库结构，包含 10 张关系表。

- **关系数据库**：MySQL 8.0+
- **向量数据库**：Qdrant（知识块向量存储）
- **ORM**：SQLAlchemy 2.0
- **字符集**：utf8mb4

---

## 2. 表清单

### 2.1 user — 用户表

存储系统所有用户账号，通过 role 区分三类角色。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, AUTO_INCREMENT | 主键 |
| account | String(64) | UNIQUE, NOT NULL | 登录账号 |
| username | String(64) | NOT NULL | 显示用户名 |
| email | String(128) | UNIQUE | 邮箱 |
| password | String(256) | NOT NULL | bcrypt 密码哈希 |
| role | String(16) | NOT NULL, DEFAULT 'student' | student / teacher / admin |
| is_active | Boolean | DEFAULT TRUE | 账号是否启用 |
| create_time | DateTime | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DateTime | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**关系**：
- 一个 user 有多个 task（学生提交作业）
- 一个 user 有多个 knowledge 记录（教师上传知识文档）
- 一个 user 有一条 config（一对一）

---

### 2.2 task — 作业任务表

每次作业提交的核心入口。`task_id`（UUID）是对外标识。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, AUTO_INCREMENT | 内部自增主键 |
| task_id | String(36) | UNIQUE, NOT NULL | 对外 UUID |
| user_id | Integer | FK → user.id | 提交作业的学生 |
| subject | String(16) | | 科目 |
| grade | String(16) | | 年级 |
| mode | String(16) | NOT NULL | 识别模式（aliyun / 模型名） |
| status | String(16) | NOT NULL | pending / processing / completed / failed |
| create_time | DateTime | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DateTime | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**状态流转**：
```
pending → processing → completed
                      ↘ failed
```

---

### 2.3 image — 作业图片表

存储作业相关的图片路径。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, AUTO_INCREMENT | 主键 |
| task_id | String(36) | FK → task.task_id | 所属任务 |
| url | String(512) | NOT NULL | MinIO 图片路径 |
| create_time | DateTime | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DateTime | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

---

### 2.4 question — 题目表

OCR 识别后的题目内容和学生答案，是批改的目标对象。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, AUTO_INCREMENT | 主键 |
| task_id | String(36) | FK → task.task_id | 所属任务 |
| no | String(8) | NOT NULL | 题号 |
| question_text | Text | | 题目文本 |
| student_answer | Text | | 学生答案 |
| question_type | String(16) | | 题目类型 |
| create_time | DateTime | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DateTime | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

---

### 2.5 block — 题目块表

阿里云切题结果，记录每道题在原图中的坐标区域。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, AUTO_INCREMENT | 主键 |
| question_id | Integer | FK → question.id | 所属题目 |
| url | String(512) | NOT NULL | 子图 MinIO 路径 |
| x1, y1 | Integer | NOT NULL | 左上角坐标 |
| x2, y2 | Integer | NOT NULL | 右下角坐标 |
| create_time | DateTime | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DateTime | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

---

### 2.6 correction — 批改结果表

每道题的得分、评语和解析。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, AUTO_INCREMENT | 主键 |
| question_id | Integer | FK → question.id, UNIQUE | 题目（一对一） |
| score | Decimal(5,2) | NOT NULL | 得分（满分 10 分） |
| result | String(16) | NOT NULL | correct / wrong / partial |
| comment | Text | | 评语 |
| analysis | Text | | 解题分析 |
| create_time | DateTime | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DateTime | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

---

### 2.7 knowledge — 知识文档表

已上传的知识文档元数据。文档经 MinerU 解析后存入 Qdrant。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, AUTO_INCREMENT | 主键 |
| user_id | Integer | FK → user.id | 上传教师 |
| title | String(256) | NOT NULL | 原始文件名 |
| url | String(512) | NOT NULL | MinIO 对象路径 |
| subject | String(16) | | 学科 |
| grade | String(16) | | 年级 |
| status | String(16) | NOT NULL | pending / parsing / completed / failed |
| chunk | Integer | DEFAULT 0 | 生成的知识块数量 |
| create_time | DateTime | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DateTime | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

---

### 2.8 question_chunk — 题目知识块关联表

批改时每题引用了哪些知识块，用于结果溯源。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, AUTO_INCREMENT | 主键 |
| question_id | Integer | FK → question.id | 题目 |
| knowledge_id | Integer | FK → knowledge.id | 来源文档 |
| chunk_id | String(36) | NOT NULL | 向量库中的块 ID |
| create_time | DateTime | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DateTime | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

---

### 2.9 config — 用户配置表

每个用户一条配置，提交作业时自动读取。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, AUTO_INCREMENT | 主键 |
| user_id | Integer | FK → user.id, UNIQUE | 用户 |
| rec_mode | String(16) | NOT NULL, DEFAULT 'aliyun' | 识别模式: aliyun / bailian |
| enable_enhance | Boolean | DEFAULT TRUE | 是否开启图片增强 |
| enable_knowledge | Boolean | DEFAULT TRUE | 是否开启知识库检索 |
| vl_model | String(64) | | 视觉模型名称 |
| gl_model | String(64) | | 批改模型名称 |
| create_time | DateTime | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | DateTime | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

---

### 2.10 model — 模型表

管理员可管理的 AI 模型列表，供用户配置时选择。

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | Integer | PK, AUTO_INCREMENT | 主键 |
| name | String(64) | NOT NULL | 模型名称 |
| mode | Integer | NOT NULL | 作用: 1=提取题目, 2=批改题目 |

---

## 3. 关系汇总

| 左表 | 关系 | 右表 | 说明 |
|------|------|------|------|
| user | 1:N | task | 一个学生提交多次作业 |
| user | 1:N | knowledge | 一个教师上传多份文档 |
| user | 1:1 | config | 一个用户有一条配置 |
| task | 1:N | image | 一次作业有多张图片 |
| task | 1:N | question | 一次作业有多道题 |
| question | 1:N | block | 一道题可能有多个区域块 |
| question | 1:1 | correction | 一道题只有一个批改结果 |
| question | M:N | knowledge | 通过 question_chunk 关联 |

---

## 4. Qdrant 向量库

### 4.1 Collection 配置

| 配置项 | 值 |
|--------|-----|
| collection_name | knowledge_chunks |
| vector_size | 1536 |
| distance | Cosine |

### 4.2 Payload 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| document_id | Integer | 关联 knowledge 表 ID |
| content | Text | 知识块文本内容（前 500 字） |
| subject | Keyword | 学科（用于过滤） |
| grade | Keyword | 年级（用于过滤） |
| seq_in_doc | Integer | 在文档中的序号 |
